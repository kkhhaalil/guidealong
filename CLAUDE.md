# CLAUDE.md — Agent handover notes

Offline, mobile-first HTML/PWA clone of the GuideAlong GPS audio tour app,
covering the Yellowstone Grand Loop with 53 stops (scenic + story/fun-fact) narrated in Mandarin.
Plain static files, no framework, no build step for the app itself — only
asset-generation scripts. Everything (map tiles, audio, Leaflet) is bundled;
the app makes **zero network requests at runtime** (verified in tests).

## Architecture

- `index.html` — single page: start overlay → map UI. Loads Leaflet, then
  `js/tour-data.js` (stops), `js/route-data.js` (route polyline), `js/app.js`.
  Registers `sw.js` at the end.
- `js/app.js` — one IIFE, no dependencies beyond Leaflet. Key parts:
  - position engine: either `watchPosition` (GPS mode) or a simulator that
    advances a fractional index along `TOUR_ROUTE` at 60 km/h × speed multiplier
    (tick = 500 ms)
  - trigger logic: haversine distance to each stop vs `stop.radius`, plus
    **direction-aware gating** — a stop only auto-triggers if it's ahead
    (bearing within `FWD_CONE`=100° of `heading`) or already within half its
    radius; `heading` is derived from consecutive position deltas. Triggered
    stops play immediately or queue behind the current narration; finished
    stops go into a `visited` Set persisted to localStorage (`ynp-tour-visited`).
    A manual preview of a distant stop is *not* marked visited, so it still
    auto-plays on arrival.
  - a WebAudio two-tone chime precedes auto-triggered narration
  - **MediaSession API**: lock-screen / car head-unit metadata + play/pause/
    next/prev/seek handlers (artwork = bundled app icons). **Screen Wake Lock**
    keeps the display awake while touring (re-acquired on `visibilitychange`).
  - **auto-resume**: sim position + speed persisted to `ynp-tour-pos`; the start
    overlay shows 「继续上次行程」 when saved progress exists; cleared by 重置.
  - **layered narration**: stops with a `more` field get a 「了解更多」 detail
    panel (transcript + season/wildlife chips + a 「播放延伸讲解」 button that
    plays `assets/audio/<id>-more.mp3`).
  - **No `fetch()`/XHR anywhere** — deliberate, so the app also works from
    `file://` (Android Chrome). Keep it that way: new data goes into JS files
    loaded via `<script>`, not JSON.
- `js/tour-data.js` — `TOUR_STOPS`: `{id, name, nameEn, lat, lng, radius,
  category, text}`. `text` is the Chinese narration script (original content);
  `id` doubles as the audio filename (`assets/audio/<id>.mp3`). Category →
  emoji icon mapping lives in `app.js` (`CAT_ICON`). A `STOP_EXTRAS` block at
  the end of the file merges optional `more` (deep-dive text), `season`, and
  `wildlife` fields by id — absent fields hide their UI automatically.
- `assets/photos/<id>.svg` — per-stop hero card (generated placeholder;
  drop in `<id>.jpg` real photos later). Shown on the now-playing card.
- `js/route-data.js` — `TOUR_ROUTE`: ~8500 `[lat, lng]` points following real
  roads, generated by OSRM. The simulator and the drawn polyline both use it.
  Lamar Valley is an out-and-back spur; the route passes Tower-Roosevelt twice.
- `sw.js` — **generated file** (by `scripts/gen_sw.py`), do not hand-edit.
  Precaches every asset (770 files, ~29 MB) in batches of 20; cache name is
  content-hashed so any asset change invalidates the old cache on next visit.

## Regeneration pipeline (scripts/)

Run from anywhere; scripts locate the repo root relative to themselves.

| Script | When to run | Notes |
|---|---|---|
| `gen_audio.py` | after editing narration text in `tour-data.js` | `pip install edge-tts` (+ Node on PATH); reads stops by evaluating `tour-data.js` with Node (so `STOP_EXTRAS` merges apply); writes `<id>.mp3` and, for stops with `more`, `<id>-more.mp3`; voice `zh-CN-YunjianNeural`, rate −4% (`TTS_VOICE`/`TTS_RATE` env to override; pass audio ids as args to regen a subset, e.g. `grand-prismatic-more`) |
| `gen_photos.py` | after adding/renaming stops | writes `assets/photos/<id>.svg` placeholder hero cards (category gradient + emoji + names); replace with real `<id>.jpg` for photography |
| `gen_route.py` | after adding/reordering stops | queries the public OSRM demo server; waypoints are hardcoded in the script — keep them in sync with `tour-data.js` order |
| `get_tiles.py` | rarely (bbox/zoom changes) | USGS National Map topo tiles (public domain), z9–12 full park bbox + z13 corridor around the route; run with `--count` first; throttled to 2 concurrent |
| `gen_icons.mjs` | icon changes | renders the icon with headless Chromium (`CHROMIUM_PATH` env to override) |
| `gen_sw.py` | **after ANY asset change** | regenerates the precache list + cache version |
| `test_app.mjs` | before committing app changes | drives the sim flow; asserts narration plays, list works, and **no external requests** (`APP_URL` env, default `http://localhost:8000`) |
| `test_pwa.mjs` | before committing SW changes | waits for full precache, then reloads with network off and asserts the app still works |

Tests need a static server running (`python3 -m http.server 8000`) and
`playwright-core` (`npm install playwright-core`) plus a Chromium binary.

## Environment gotchas (Claude Code cloud sandbox)

- Outbound TLS is re-terminated by a proxy. Python `urllib`/`edge-tts` need the
  CA bundle: scripts accept `TILE_CA_BUNDLE=/root/.ccr/ca-bundle.crt`; for
  edge-tts append that bundle to certifi's store
  (`cat /root/.ccr/ca-bundle.crt >> $(python3 -c "import certifi; print(certifi.where())")`).
- **openstreetmap.org tile server blocks bulk downloads** (serves "Access
  blocked" placeholder tiles). Don't retry OSM; that's why tiles come from USGS.
- **cloudflared tunnels do not work here** — the egress proxy only allows
  HTTPS on 443, and Cloudflare's tunnel transport uses port 7844 (QUIC + HTTP/2).
  Confirmed failing 2026-07: `dial tcp :7844: i/o timeout`. Don't waste time
  re-trying; use a real HTTPS host instead.
- GitHub access is repo-scoped; downloading release binaries from other GitHub
  repos fails. cloudflared (if ever needed elsewhere) is also on
  `pkg.cloudflare.com`.

## Deployment status / open item

The repo is currently **private**, and Pages on a private repo requires a paid
GitHub plan (`has_pages: false` as of 2026-07-02). To publish the PWA:
either make the repo public and enable GitHub Pages (root of the deployed
branch works as-is — all app URLs are relative, so a `/guidealong/` subpath is
fine), or use any other HTTPS static host (Netlify/Vercel drop the folder in
as-is). After first HTTPS load, the service worker makes it permanently
offline-capable and installable.

## Content notes

- Narration scripts are original text written for this project (facts about
  Yellowstone from general knowledge, not verified against live sources —
  flagged to the user already). Style: warm tour-guide register, 您-form,
  ~150–230 chars per stop (≈45–60 s spoken).
- Tile attribution ("USGS The National Map") is shown in the Leaflet
  attribution control; route data derives from OpenStreetMap via OSRM (ODbL) —
  keep the README credits if these change.
- UI language is Chinese throughout; `nameEn` fields are shown as secondary
  labels in the stop list.
